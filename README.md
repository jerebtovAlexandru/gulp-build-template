# Gulp Build Template 🚀

Современный, быстрый и гибко настраиваемый стартовый шаблон (Starter Pack) для эффективной веб-разработки и автоматизации рутинных задач front-end разработчика.

## 📋 Суть и описание проекта

Этот проект представляет собой готовую среду сборки, которая берет на себя всю грязную работу: компиляцию препроцессоров, оптимизацию графики, настройку локального сервера и подготовку проекта к деплою на реальный хостинг. 

Сборка построена на базе современного **Gulp 5** с использованием **ECMAScript модулей (ESM)**, что обеспечивает высокую скорость работы и поддержку актуальных стандартов JavaScript.

---

## 🛠 Технологический стек (Built With)

*   **Автоматизация:** [Gulp 5](https://gulpjs.com) (в связке с `gulp-if`, `gulp-plumber`, `gulp-notify`)
*   **Стили:** [Sass/SCSS](https://sass-lang.com) (`dart-sass`, `gulp-autoprefixer`, `gulp-clean-css`, `gulp-group-css-media-queries`)
*   **Скрипты:** [Webpack](https://js.org) (`webpack-stream` для сборки JS-модулей в единый файл)
*   **Разметка:** HTML5 (`gulp-file-include` для модульной сборки страниц)
*   **Графика и Шрифты:** `gulp-sharp-responsive` (умный ресайз), `gulp-webp`, `gulp-imagemin`, `gulp-svg-sprite`, `gulp-fonter`, `ttf2woff2`
*   **Инструменты:** `browser-sync` (локальный сервер), `vinyl-ftp` (авто-деплой), `gulp-zip` (архивация)

## 🚀 Инструкция по установке и запуску

### 1. Требования
Убедитесь, что на вашем компьютере установлена [Node.js](https://nodejs.org) (рекомендуется LTS версия).

### 2. Клонирование репозитория
Склонируйте проект к себе на компьютер:
```bash
git clone https://github.com
cd gulp-build-template
```

### 3. Установка зависимостей
Установите все необходимые npm-пакеты, прописанные в `package.json`:
```bash
npm install
```

## 💻 Как пользоваться (Команды терминала)

В проекте настроены удобные npm-скрипты для управления сборкой:

| Команда | Что делает? |
| :--- | :--- |
| `npm run dev` | Запускает режим разработки: компиляцию, локальный сервер на порту `7272` и вотчер (слежение за файлами). |
| `npm run build` | Создает финальную, оптимизированную и сжатую сборку проекта в папке `/dist`. |
| `npm run svgSprite` | Ручной запуск сборки единого SVG-спрайта из иконок в папке `src/svgicons`. |
| `npm run zip` | Автоматически собирает проект и упаковывает папку `/dist` в ZIP-архив. |
| `npm run ftp` | Полная сборка проекта с последующей автоматической выгрузкой на удаленный сервер по FTP. |

> **Важно для FTP:** Перед использованием команды `npm run ftp` обязательно заполните свои доступы (хост, логин, пароль) в файле `gulp/config/ftp.js`.

## 📂 Структура папок проекта

```text
├── gulp/                  # Конфигурация и отдельные задачи Gulp
│   ├── config/            # Настройки путей, плагинов и FTP
│   └── tasks/             # Изолированные таски (html, scss, js, изображения, шрифты)
├── src/                   # Исходные файлы разработки (Source)
│   ├── files/             # Статичные файлы (копируются без изменений)
│   ├── img/               # Картинки проекта
│   ├── js/                # JavaScript (точка входа app.js)
│   ├── scss/              # Стили SCSS (главный файл style.scss)
│   ├── svgicons/          # Иконки для генерации спрайта
│   └── *.html             # Страницы HTML
├── dist/                  # Папка готовой сборки (появляется после запуска, в Git не коммитится)
├── gulpfile.js            # Главный конфигурационный файл Gulp
├── package.json           # Зависимости и скрипты проекта
└── README.md              # Документация проекта
```

## 💡 Особенности сборки

1.  **Cache Busting (Сброс кэша):** В режиме `build` к путям подключения CSS и JS файлов автоматически добавляется уникальный временной хэш вида `style.min.css?_v=XXXXXXXX`. Это гарантирует, что клиенты увидят обновления сайта мгновенно, минуя кэш браузера.
2.  **Next-Gen Graphics (WebP):** Сборка автоматически конвертирует ваши `.jpg` и `.png` изображения в современный формат `.webp`, а плагин в HTML заменяет обычные теги `<img>` на адаптивную конструкцию `<picture>`.
3.  **Умный ресайз картинок:** Если вы закинете в проект огромную картинку, плагин `sharp-responsive` автоматически уменьшит её максимальную ширину до `1920px`, чтобы не перегружать сайт (маленькие картинки увеличиваться не будут).
4.  **Модульный HTML:** Вы можете разделять верстку на независимые блоки (например, `@@include('header.html')`) для повторного использования кода.
5.  **Автоматические шрифты:** Достаточно просто закинуть шрифты в форматах `.otf` или `.ttf` в исходники — сборка сама переведет их в веб-форматы `.woff`/`.woff2` и самостоятельно напишет для них файл `fonts.scss` со всеми правилами `@font-face` и корректным `font-weight`.

## 🔧 Настройка окружения и фичи сборки

Для того чтобы в вашем редакторе VS Code корректно работало автодополнение путей (умные подсказки при вводе `@img/` или `@scss/`), а сборщик правильно заменял эти пути при компиляции, выполните следующие шаги.

### 1. Настройка автодополнения путей в VS Code (Path Autocomplete)

1. Установите в VS Code расширение **Path Autocomplete** (автор: *Mihai临时/Ionut Mihai*).
2. Откройте глобальные настройки редактора в формате JSON. Для этого нажмите `Ctrl + Shift + P` (или `Cmd + Shift + P` на Mac) и выберите пункт:
   **Preferences: Open User Settings (JSON)** (Открыть настройки пользователя).
3. Добавьте в открывшийся файл конфигурации следующий блок сопоставления путей (алиасов):

```json
"path-autocomplete.pathMappings": {
  "@img": "\${folder}/src/img",   // Алиас для быстрого поиска изображений
  "@scss": "\${folder}/src/scss", // Алиас для стилей препроцессора
  "@js": "\${folder}/src/js"      // Алиас для JavaScript модулей
}
```

### 2. Умная замена путей (Работа плагина `gulp-replace`)

Благодаря настроенному алиасу, вы можете писать пути к картинкам в стилях (SCSS) и разметке (HTML) через удобный короткий префикс `@img/`. 

*   **Как вы пишете в коде:** `<img src="@img/avatar.png">` или `background-image: url('@img/bg.jpg')`
*   **Как это работает:** В процессе сборки таски `html.js` и `scss.js` пропускают код через плагин `gulp-replace`, который на лету находит регулярное выражение `@img/` и заменяет его на физически корректный путь для папки назначения (`img/` в HTML и `../img/` в CSS).

### 3. Модульная сборка страниц (`gulp-file-include`)

Для разделения HTML на независимые переиспользуемые блоки (компоненты) используется плагин `gulp-file-include`. Вы можете передавать динамические переменные (например, заголовок страницы `title`) прямо внутрь подключаемого шаблона.

**Пример структуры главной страницы (`src/index.html`):**

```html
<!DOCTYPE html>
<html lang="ru">
@@include('html/head.html', {'title': 'Главная'})
<body>
   @@include('html/header.html', {'title': 'Главная'})
   
   <main>
      <!-- Контент страницы -->
      <img src="@img/hero-bg.jpg" alt="Фон">
   </main>
   
   @@include('html/footer.html', {'title': 'Главная'})
   <script src="js/app.min.js"></script>
</body>
</html>
```

*Обратите внимание: папка `html/` с компонентами (`head.html`, `header.html`, `footer.html`) должна находиться внутри вашей директории `src/`.*


## 📦 Описание всех зависимостей (devDependencies)

Ниже представлен полный список всех установленных npm-пакетов, которые обеспечивают автоматизацию, сборку и оптимизацию проекта.

<details>
<summary>📂 Кликните, чтобы развернуть и посмотреть назначение каждого плагина</summary>

### 🌐 Сервер и автоматизация
*   `browser-sync` — локальный сервер с автоматической перезагрузкой страниц в браузере при изменении кода.
*   `del` — плагин для полной очистки папки готовой сборки (`/dist`) перед новым запуском тасков.
*   `gulp` — ядро сборщика, управляющее всеми потоками файлов и сценариями автоматизации.
*   `gulp-if` — логический оператор условий, позволяющий разделять сборку на режим разработки (`dev`) и продакшена (`build`).
*   `gulp-plumber` — перехватчик ошибок в потоках, предотвращающий падение и остановку Gulp при ошибках в коде.
*   `gulp-notify` — вывод красивых всплывающих окон с уведомлениями об ошибках на рабочий стол.

### 🎨 Стили (CSS / SCSS)
*   `sass` — основной компилятор препроцессора Sass.
*   `gulp-sass` — плагин для интеграции компилятора Sass в потоки задач Gulp.
*   `gulp-autoprefixer` — автоматическое добавление вендорных префиксов (`-webkit-`, `-moz-`) для поддержки старых браузеров.
*   `gulp-clean-css` — глубокое сжатие, минификация и оптимизация готовых файлов стилей.
*   `gulp-group-css-media-queries` — сбор всех разрозненных медиа-запросов (`@media`) и их группировка в самом конце CSS-файла.

### 🔤 Шрифты
*   `fonteditor-core` — низкоуровневое ядро для парсинга, анализа и конвертации шрифтов.
*   `fs` — заглушка безопасности для встроенного системного модуля Node.js, работающего с файловой системой.
*   `gulp-fonter` — конвертер шрифтов из формата TTF в форматы EOT, WOFF и OTF.
*   `gulp-ttf2woff` — специализированный плагин для конвертации шрифтов TTF в веб-формат WOFF.
*   `gulp-ttf2woff2` — плагин для быстрой конвертации TTF в самый современный и сжатый веб-формат WOFF2.

### 🖼 Изображения и иконки
*   `gulp-sharp-responsive` — мощный плагин для изменения размеров изображений под разные экраны (адаптивный ресайз).
*   `gulp-webp` — автоматическая конвертация растровой графики (JPG/PNG) в современный легковесный формат WebP.
*   `gulp-webpcss` — добавление классов поддержки WebP (`.webp` / `.no-webp`) прямо внутрь CSS для фоновых изображений (`background-image`).
*   `gulp-webp-html-nosvg` — автоматическая обертка тегов `<img>` в HTML в адаптивную конструкцию `<picture>` с WebP версиями.
*   `gulp-imagemin` — глубокое сжатие и оптимизация растровых изображений без потери качества.
*   `gulp-svg-sprite` — объединение множества отдельных SVG-иконок в один оптимизированный SVG-спрайт.
*   `webp-converter` — вспомогательный движок (инструмент) для кодирования изображений в формат WebP.

### 📝 Разметка и скрипты
*   `gulp-file-include` — модульная сборка HTML, позволяющая собирать страницы из отдельных компонентов (шапка, подвал).
*   `gulp-replace` — инструмент поиска и автоматической замены строк (используется для работы с алиасами `@img`).
*   `gulp-rename` — переименование выходных файлов (создание файлов с суффиксом `.min.css` и `.min.js`).
*   `webpack` / `webpack-stream` — мощный сборщик JavaScript-модулей, минификатор и оптимизатор скриптов.

### 🚀 Деплой и архивация
*   `gulp-version-number` — добавление уникального хэша версии к путям CSS/JS файлов для сброса кэша в браузере (Cache Busting).
*   `gulp-zip` — автоматическая упаковка готовой папки дистрибутива в ZIP-архив.
*   `vinyl-ftp` — модуль для быстрой и автоматической выгрузки собранного проекта на удаленный хостинг по FTP-соединению.
*   `gulp-util` — набор вспомогательных утилит для логирования процессов сборки в консоль.
